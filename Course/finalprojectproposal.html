Aesthetic Goals

Swiping left or right through "views" or "pages" of data has become standard in todays modern user interfaces. My artistic goal for this project involves becoming fluent with manipulating the properties(such as alpha X/Y/Z, scale, rotation etc...) of each page. This project will specifically focus on animating the elements inside the page.

Technical Goals

To accomplish my Aesthetic goal I will seek to become familiar in the most basic sense with Google's "Viewpager" implementation

to use as a model (or and other slide-show-esque web implementations). I will also have to become familiar with translating each page's elements relative to the visible page and the adjacent pages just off the screen.

Sketch/Diagram

This project will be preferably work only on mobile devices(as some examples on p5.js are seen to require mobile's to work). The front-end will consist of a 2 or 4 setup, each taking up the full screen. Each page will have its own elements that, upon swipe initiation, will be exposed to a common interface that will provide positional coordinates that encompass the visible and adjacent pages just off the screen. **EDIT: While typing this I discovered that there is no real notion of "interfaces" in Javascript as there is in Java because Javascript uses what is called duck typing.

With the provided positional parameters I will then be able to manipulate elements inside each page according to the state of the current page transition("swipe") that is occuring.
